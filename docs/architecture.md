# 架构草案

## 核心模块

- QQ 接入层：连接 QQ 协议端或 OneBot 兼容服务，接收私聊和群聊消息。
- 消息处理层：过滤命令、权限、黑名单、触发词和上下文。
- AI 调用层：向大模型服务发送请求并处理回复。
- 记忆存储层：保存用户会话、群聊上下文、会话摘要和长期回忆摘要。
- 日志监控层：记录运行状态、错误和关键事件。

## 基本流程

1. QQ 协议端收到消息。
2. 机器人框架把消息转发给本项目。
3. 项目判断是否需要 AI 回复。
4. 项目拼接系统提示词、历史上下文和当前消息。
5. 调用 AI 接口生成回复。
6. 将回复发送回 QQ。
7. 保存日志和必要的上下文。

## 可选技术路线

- Python: NoneBot2 + OneBot v11/v12 + OpenAI SDK。
- Node.js: Koishi 或自定义 OneBot 客户端 + AI SDK。
- 本地模型: Ollama、vLLM 或 LM Studio 作为模型服务。
