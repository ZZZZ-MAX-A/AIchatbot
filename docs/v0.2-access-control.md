# v0.2 权限与安全设计

## 目标

v0.2 的目标是让 QQ AI 机器人在私聊和群聊中更安全、可控、可长期运行。

第一批实现范围：

- 主人权限
- 私聊白名单
- 陌生人私聊试用开关
- 陌生人私聊最多 3 次 AI 回复
- 群聊白名单
- 用户黑名单
- 用户级冷却
- 私聊消息长度限制 150 字
- 群聊消息长度限制 300 字
- AI 错误日志追加写入

## 采用方案

采用方案 A：严格安全型。

核心规则：

- 主人私聊无限制。
- 私聊白名单用户可以私聊。
- 普通用户默认不能私聊。
- 如果开启陌生人私聊试用，普通用户最多获得 3 次 AI 回复。
- 群聊只允许白名单群使用。
- 白名单群内所有非黑名单用户都可以通过 @机器人 触发回复。
- 黑名单用户私聊和群聊都不响应。
- 同一用户触发 AI 回复有冷却时间。
- 超过长度限制的消息直接拒绝，不自动截断。

## 身份模型

### Owner 主人

由 `BOT_OWNER_QQ` 配置。

权限：

- 可以私聊机器人。
- 不受普通私聊白名单限制。
- 可以执行后续管理命令。
- 仍然受基础安全限制，例如单条消息长度限制可后续单独放宽。

### Private Whitelist 私聊白名单用户

由 `PRIVATE_WHITELIST` 配置。

权限：

- 可以私聊机器人。
- 受私聊冷却限制。
- 受私聊消息长度限制。

### Group Whitelist 群白名单

由 `GROUP_WHITELIST` 配置。

规则：

- 只有白名单群允许群聊 AI 回复。
- 白名单群内所有非黑名单用户都可以 @机器人使用。

### Blocked User 黑名单用户

由 `USER_BLACKLIST` 配置。

规则：

- 私聊不响应。
- 群聊不响应。
- 后续管理命令也不响应。

### Normal User 普通用户

规则：

- 默认不能私聊。
- 如果开启 `ALLOW_UNKNOWN_PRIVATE_CHAT=true`，可以试用私聊。
- 试用次数由 `PRIVATE_TRIAL_MESSAGES` 控制，默认 3 次 AI 回复。
- 在白名单群内可以 @机器人聊天。

## 私聊规则

私聊处理顺序：

1. 如果用户在 `USER_BLACKLIST` 中，静默不回复。
2. 如果用户是 `BOT_OWNER_QQ`，允许。
3. 如果用户在 `PRIVATE_WHITELIST` 中，允许。
4. 如果 `ALLOW_UNKNOWN_PRIVATE_CHAT=true`，进入试用逻辑。
5. 否则拒绝私聊。

私聊长度限制：

```text
MAX_PRIVATE_MESSAGE_LENGTH=150
```

超过限制时回复：

```text
消息太长了，请控制在 150 字以内。
```

陌生人私聊关闭时回复：

```text
当前不开放私聊，请在授权群聊中 @我使用。
```

陌生人试用次数用完时回复：

```text
私聊试用次数已用完，请联系主人加入白名单。
```

## 群聊规则

群聊处理顺序：

1. 必须 @机器人。
2. 如果用户在 `USER_BLACKLIST` 中，静默不回复。
3. 如果 `ENABLE_GROUP_CHAT=false`，静默不回复。
4. 如果 `GROUP_WHITELIST` 为空，默认不开放群聊。
5. 如果当前群不在 `GROUP_WHITELIST` 中，静默不回复。
6. 检查用户冷却。
7. 检查消息长度。
8. 调用 AI 回复。

群聊长度限制：

```text
MAX_GROUP_MESSAGE_LENGTH=300
```

超过限制时回复：

```text
消息太长了，请控制在 300 字以内。
```

未授权群：

```text
静默，不回复。
```

## 冷却规则

冷却按用户维度计算。

配置：

```env
PRIVATE_RATE_LIMIT_SECONDS=10
GROUP_RATE_LIMIT_SECONDS=5
```

含义：

- 同一用户私聊 10 秒内只能触发一次 AI 回复。
- 同一用户在群聊 5 秒内只能触发一次 AI 回复。

冷却中回复：

```text
说太快了，稍等几秒再试。
```

## 配置项

`.env` 示例：

```env
BOT_OWNER_QQ=3313097998

ENABLE_PRIVATE_CHAT=true
ENABLE_GROUP_CHAT=true

PRIVATE_WHITELIST=
ALLOW_UNKNOWN_PRIVATE_CHAT=false
PRIVATE_TRIAL_MESSAGES=3
PRIVATE_RATE_LIMIT_SECONDS=10
MAX_PRIVATE_MESSAGE_LENGTH=150

GROUP_WHITELIST=
GROUP_RATE_LIMIT_SECONDS=5
MAX_GROUP_MESSAGE_LENGTH=300

USER_BLACKLIST=
```

说明：

- `GROUP_WHITELIST` 为空时，群聊默认不开放。
- `PRIVATE_WHITELIST` 为空时，只有主人可以私聊。
- 多个 QQ 号或群号使用英文逗号分隔。

示例：

```env
BOT_OWNER_QQ=3313097998
PRIVATE_WHITELIST=123456789,987654321
GROUP_WHITELIST=111111111,222222222
USER_BLACKLIST=999999999
```

## 命令规划

v0.2 第一批保留：

```text
/status
/reset
```

`/status` 增强显示：

- 当前模型
- 私聊是否开启
- 群聊是否开启
- 私聊白名单数量
- 群白名单数量
- 黑名单数量
- 私聊长度限制
- 群聊长度限制
- 私聊冷却
- 群聊冷却

v0.2 第二批管理命令：

```text
/启用本群
/禁用本群
/加入群白名单 群号
/移出群白名单 群号
/加入私聊白名单 QQ号
/移出私聊白名单 QQ号
/加入黑名单 QQ号
/移出黑名单 QQ号
/群白名单
/私聊白名单
/黑名单
```

所有管理命令只允许 `BOT_OWNER_QQ` 使用。

动态名单保存到：

```text
data/access.json
```

最终权限 = `.env` 静态名单 + `data/access.json` 动态名单。

## 错误日志

AI 调用失败时追加写入：

```text
logs/ai_chat_error.log
```

日志内容：

- 时间
- 会话类型
- 用户 QQ
- 群号，如果有
- 异常类型
- 异常消息

不记录：

- API Key
- Token
- 完整私密配置

## 测试用例

### 私聊

1. 主人私聊机器人，应该正常回复。
2. 私聊白名单用户私聊机器人，应该正常回复。
3. 普通用户私聊，`ALLOW_UNKNOWN_PRIVATE_CHAT=false`，应该收到拒绝提示。
4. 普通用户私聊，`ALLOW_UNKNOWN_PRIVATE_CHAT=true`，最多获得 3 次 AI 回复。
5. 普通用户试用超过 3 次，应该收到试用结束提示。
6. 黑名单用户私聊，应该静默。
7. 私聊消息超过 150 字，应该拒绝。
8. 同一用户短时间连续私聊，应该触发冷却提示。

### 群聊

1. 白名单群内 @机器人，应该正常回复。
2. 非白名单群内 @机器人，应该静默。
3. 白名单群内黑名单用户 @机器人，应该静默。
4. 群聊消息超过 300 字，应该拒绝。
5. 同一用户短时间连续 @机器人，应该触发冷却提示。
6. 没有 @机器人，应该不回复。

### 配置

1. 修改 `.env` 后重启机器人，配置应该生效。
2. `GROUP_WHITELIST` 为空时，群聊默认不开放。
3. `PRIVATE_WHITELIST` 为空时，只有主人可以私聊。
