# v0.5 长期回忆摘要

## 目标

v0.5 实现长期回忆摘要的第一批能力：由主人手动给当前对话对象添加、查看、删除和清空长期回忆摘要。

当前阶段不做 AI 自动提取，原因是：

- 手动摘要可控，不容易把闲聊和误判写入长期回忆。
- 可以先验证长期回忆表、命令、上下文注入是否稳定。
- 后续做自动提取时，可以复用这一套查看和删除能力。

## 记忆边界

长期回忆摘要回答的是：

```text
当前对话对象有哪些需要长期记住的大致背景？
```

它不同于短期记忆和会话摘要：

```text
短期记忆：最近正在聊什么
会话摘要：较早的一段聊天大概说过什么
长期回忆摘要：当前对话对象值得长期保留的大致背景
```

例子：

```text
短期记忆：刚刚讨论了摘要压缩命令。
会话摘要：这段对话完成了 v0.4 摘要压缩测试。
长期回忆摘要：当前对话对象正在推进 QQ AI 机器人项目，已完成短期记忆和摘要压缩，当前阶段关注长期回忆摘要与摘要系统融合。
```

## QQ 命令

以下命令只有主人可以使用。

### 添加当前对象回忆摘要

私聊中，当前对象是私聊用户。

群聊中，当前对象是当前群。

```text
/添加记忆 内容
```

例子：

```text
/添加记忆 当前对话对象正在测试 QQ AI 机器人的长期回忆摘要功能，已确认短期记忆和摘要压缩可用。
```

### 重写当前对象回忆摘要

```text
/重写当前记忆 内容
```

这个命令会先清空当前对象所有长期回忆摘要，再写入一条新的长期回忆摘要。

它适合在长期回忆摘要变散、变旧或需要合并时使用。

例子：

```text
/重写当前记忆 当前对话对象正在开发 QQ AI 机器人，已完成权限、短期记忆和摘要压缩，目前正在测试长期回忆摘要。
```

### 查看记忆

查看当前对象记忆：

```text
/查看记忆
```

### 删除记忆

```text
/删除记忆 记忆ID
```

记忆 ID 可以从 `/查看记忆` 中看到。

### 清空当前记忆

```text
/清空当前记忆
```

私聊中会清空当前用户长期回忆摘要。

群聊中会清空当前群长期回忆摘要。

为避免旧信息从其他记忆层继续影响回复，当前实现还会同时清空：

```text
当前会话摘要
当前短期上下文
```

## 上下文注入

每次调用 AI 时，机器人会读取和当前对话相关的长期记忆：

```text
私聊：当前用户长期回忆摘要
群聊：当前群长期回忆摘要
```

默认最多带入：

```env
MAX_LONG_TERM_MEMORIES_IN_CONTEXT=8
```

长期回忆摘要会作为 system 上下文传给模型，提示模型“可以参考，但不要编造额外事实”。

实际上下文提示会要求模型：

```text
仅在与当前问题相关时参考长期回忆摘要
不要强行提起长期回忆摘要
不要编造额外事实
```

## 隐私

长期记忆保存在：

```text
data/chatbot.db
```

这个文件已经被 `.gitignore` 忽略，不会提交到 GitHub。

长期回忆摘要可能包含真实对话背景或项目情况，添加前应避免写入：

```text
API Key
Token
二维码链接
账号密码
身份证、手机号等敏感个人信息
```

## 测试计划

基础测试：

1. 私聊发送 `/添加记忆 当前对象正在测试长期回忆摘要。`
2. 私聊发送 `/查看记忆`，确认能看到新增记忆和 ID。
3. 继续聊天，询问机器人是否记得当前对象的长期回忆摘要。
4. 发送 `/重写当前记忆 当前对象长期回忆摘要已被重写。`，确认旧摘要被替换。
5. 发送 `/删除记忆 ID`，确认能删除。
6. 群聊发送 `/添加记忆 本群正在测试长期回忆摘要。`
7. 群聊发送 `/查看记忆`，确认保存到当前群。
8. 发送 `/清空当前记忆`，确认当前对象长期记忆、会话摘要和短期上下文被清空。
9. 非主人执行这些命令应被拒绝。

回归测试：

1. 短期记忆仍然正常。
2. 摘要压缩仍然正常。
3. `/记忆状态` 能显示长期记忆数量。
4. 白名单、黑名单和权限命令不受影响。
