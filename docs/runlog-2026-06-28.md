## 今日启动相关结论

### Ollama 视觉服务

图片识别依赖本地 Ollama：

```env
VISION_OLLAMA_BASE_URL=http://127.0.0.1:11434
VISION_MODEL=qwen2.5vl:3b
```

今天发现并确认一个关键问题：

如果直接让托盘版 `ollama app.exe` 接管 11434，视觉模型可能从中文用户目录加载：

```text
C:\Users\雨泽\.ollama\models
```

Ollama / llama-server 在加载视觉 CLIP/mmproj 时可能把路径读成乱码：

```text
C:\Users\ÓêÔó\.olla...
```

QQ 内机器人表现为：

```text
Ollama HTTP 500
llama-server process has terminated
Failed to load CLIP model ...
```

这不是图片代码链路问题，而是 Ollama 视觉模型加载路径问题。

已经新增固定启动脚本：

```powershell
.\scripts\start-ollama-vision.ps1
```

该脚本会：

- 关闭现有 `ollama.exe` / `ollama app.exe`
- 设置 `OLLAMA_MODELS=D:\OllamaModels`
- 用 `ollama serve` 启动 11434 服务
- 检查 `/api/tags`

如果要写入当前 Windows 用户环境变量：

```powershell
.\scripts\start-ollama-vision.ps1 -PersistUserEnv
```

今天已用最小 PNG 验证：

```json
{
  "ok": true,
  "elapsed_seconds": 9.391,
  "result": "图片中显示的是一个纯白色的背景。没有其他物体或内容可以描述。"
}
```

结论：正确用 `D:\OllamaModels` 启动后，Ollama 视觉模型能正常加载并返回描述。

### 推荐日常启动顺序

需要图片识别时：
