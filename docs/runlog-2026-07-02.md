## ActionRequest schema

当前 MainAgentGraph 已有结构化 action 请求：

```text
MainAgentAction
  final_answer
  tool_request
  ask_owner
  stop

MainAgentActionRequest
  action
  content
  tool_name
  arguments
  reason
```

当前只允许的工具：

```text
dev_context
```

当前 parser 会拒绝：

```text
非法 JSON
非 object payload
空 action
未知 action
final_answer 缺 content
ask_owner 缺 content
tool_request 缺 tool_name
tool_request 请求非 dev_context 工具
dev_context 缺 arguments.query
非 tool_request action 携带 tool_name
```

当前 CALL_MAIN_AGENT stub 行为：

```text
call_main_agent
  -> dev_context_tool_action_json(query, reason="恢复开发侧项目上下文")
  -> raw_action_request 为 JSON 字符串

validate_action_request
  -> parse_main_agent_action_request
  -> apply_action_request_to_state
```

也就是说，真实 Main LLM 未来只需要替换 `call_main_agent` 的生成逻辑，但输出仍必须经过同一套 ActionRequest schema、ToolPolicyCheck 和工具执行边界。
