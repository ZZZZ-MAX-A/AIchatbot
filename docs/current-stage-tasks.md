# 当前阶段任务文档

## 阶段定位

当前阶段的核心目标是把机器人的记忆系统做稳，而不是继续扩展复杂人格、偏好或自动推理功能。

当前记忆系统分为三块：

```text
短期记忆：保存最近对话原文
会话摘要：压缩较旧的聊天记录
长期回忆摘要：保存当前对话对象值得长期保留的大致摘要
```

当前阶段重点：

```text
让 AI 能在回复时参考当前对话对象的长期回忆摘要
让长期回忆摘要和会话摘要共同参与上下文
保持主人手动控制长期记忆内容
暂不做 AI 自动提取长期记忆
暂不做偏好、性格、设定等细分类管理
暂不做语义向量索引
```

## 当前阶段设计原则

### 1. 长期记忆只保存摘要

当前不做细粒度偏好管理。

长期回忆摘要应该像这样：

```text
当前对象正在测试 QQ AI 机器人的长期回忆摘要功能，已确认短期记忆和摘要压缩可用，目前重点是让 AI 能结合当前对象的历史摘要进行回复。
```

不建议写成：

```text
用户喜欢简短回复。
用户性格焦虑。
本群设定为开发群。
```

这些可以以后再做，但当前阶段不实现。

### 2. 摘要必须客观

长期回忆摘要和会话摘要都不能编造。

可以记录：

```text
已经完成的功能
当前正在讨论的方向
主人明确确认过的方案
当前对象相关的重要背景
```

不应该记录：

```text
主观猜测
没有明确表达过的动机
敏感信息
过细的性格判断
临时情绪
```

### 3. 主人手动控制

当前阶段不让 AI 自动写长期记忆。

原因：

```text
避免 AI 把无关闲聊写成长期记忆
避免错误总结污染记忆库
先把查看、删除、清空和上下文参与流程跑稳
```

## 当前需要测试的内容

### 测试 1：私聊长期回忆摘要

在主人私聊机器人时发送：

```text
/添加记忆 当前对象正在测试长期回忆摘要功能，已确认短期记忆和摘要压缩可用。
```

然后发送：

```text
/查看记忆
```

```text
机器人返回刚刚添加的长期回忆摘要
```


## 当前阶段验收标准

满足以下条件即可认为当前阶段完成：

```text
私聊可以添加、查看、删除、清空长期回忆摘要
群聊可以添加、查看、删除、清空当前群长期回忆摘要
长期回忆摘要能参与 AI 回复
非主人不能管理长期回忆摘要
短期记忆不受影响
会话摘要压缩不受影响
/记忆状态 能看到长期记忆数量变化
/记忆状态 能看到当前对象长期回忆摘要数量
数据库文件仍然不会提交到 GitHub
```

## 暂不做的功能

当前阶段明确不做：

```text
AI 自动提取长期记忆
偏好分类
性格分类
用户画像
机器人拟人化
语义向量索引
跨对象主动关联记忆
按 QQ 号或群号远程添加记忆
```

这些能力后续可以做，但必须在长期回忆摘要稳定后再考虑。
