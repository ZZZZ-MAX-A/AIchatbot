# v0.8 主人通知与转告方案

## 阶段定位

v0.8 的目标不是让 AI 自由主动聊天，而是先实现一个可控的主人通知模块。

第一版只做一件事：

```text
允许授权用户用明确命令向主人发送短文本转告。
```

当前不做：

```text
AI 自主判断是否主动找主人聊天
随机主动聊天
定时问候
自然语言自动识别转告
AI 改写或润色转告内容
图片、语音、文件、二维码转告
主人回复后自动转回群聊
```

## 核心命令

已实现命令：

```text
/转告主人 内容
/留言给主人 内容
```

两个命令行为一致。

示例：

```text
/转告主人 服务器异常，请主人看群。
/留言给主人 明天十点开会。
```

## 权限边界

允许触发转告的来源：

```text
主人本人
私聊白名单用户
授权群里的群成员
```

禁止触发转告的来源：

```text
黑名单用户
非授权群
陌生私聊用户
```

第一版不允许任何人用自然语言触发转告，只认明确命令。

## 内容限制

转告内容必须满足：

```text
不能为空
纯文本
最多 50 字
不包含明显敏感信息
```

超过 50 字时固定回复：

```text
请主动联系主人，文本过长不予转告。
```

不要自动截断。

不要让 AI 帮忙摘要。

不要让 AI 润色或改写。

原因：

```text
转告需要保真
截断可能改变语义
摘要可能遗漏或扭曲重点
长文本容易夹带隐私、争执或诱导内容
```

空内容时建议回复：

```text
请填写 50 字以内的转告内容。
```

## 敏感信息拦截

即使内容不超过 50 字，也应拦截明显敏感信息。

第一版建议拦截：

```text
密码
验证码
Token
API Key
身份证号
手机号
二维码
数据库内容
```

命中敏感内容时回复：

```text
内容包含敏感信息，不予转告。
```

边界：

```text
危险或不合理请求本身可以转告给主人判断
敏感数据本身不转告
```

例如：

```text
/转告主人 有人想让主人发 token
```

可以转告。

```text
/转告主人 token 是 xxx
```

不转告。

## 转告格式

转告给主人时不调用 LLM，不走角色卡。

群聊来源格式：

```text
收到一条转告：
来源：群 群号
发送人：QQ号
内容：转告内容
```

私聊来源格式：

```text
收到一条转告：
来源：私聊用户 QQ号
内容：转告内容
```

原会话成功提示：

```text
已转告主人。
```

发送失败提示：

```text
转告发送失败，请稍后再试。
```

不要暴露主人在线状态、私聊状态或更多内部错误细节。

## 冷却限制

必须限制频率，避免机器人变成骚扰入口。

第一版冷却：

```text
同一用户：300 秒一次
同一群：120 秒一次
全局给主人：60 秒一次
```

冷却期间回复：

```text
转告过于频繁，请稍后再试。
```

配置项：

```env
ENABLE_OWNER_NOTIFICATIONS=true
OWNER_NOTIFICATION_MAX_LENGTH=50
OWNER_NOTIFICATION_GLOBAL_COOLDOWN_SECONDS=60
OWNER_NOTIFICATION_GROUP_COOLDOWN_SECONDS=120
OWNER_NOTIFICATION_USER_COOLDOWN_SECONDS=300
```

## 数据记录

第一版发送失败会写入现有错误日志。

如果需要更好排查，后续可新增表：

```text
owner_notifications
```

字段建议：

```text
id
source_type
source_id
sender_user_id
content
status
reason
created_at
sent_at
```

当前不提供 QQ 内查询命令。

后续可考虑：

```text
/查看转告记录
/清空转告记录
```

## 和现有模块的关系

### 和角色卡

转告内容不走角色卡，不由 AI 改写。

原因：

```text
角色卡可能改变语气
角色卡可能添加称谓或情绪
转告最重要的是准确和可追溯
```

### 和摘要

原会话中的 `/转告主人 内容` 会作为普通用户消息进入短期上下文。

机器人回复的“已转告主人。”也可以进入短期上下文。

发送给主人私聊的通知内容不写入原群聊会话。

主人私聊收到通知后的回应，不自动转回原群。

### 和长期记忆

长期记忆模块当前暂停使用。

转告功能不写长期记忆。

## 主动聊天边界

第一版不做 AI 主动陪聊。

允许的主动行为只有：

```text
收到明确转告命令后，主动给主人发固定格式通知。
```

后续可单独设计：

```text
系统错误通知主人
白名单申请通知主人
摘要压缩失败通知主人
群内高风险事件通知主人
主人订阅型定时提醒
```

不建议直接实现：

```text
AI 随机找主人聊天
AI 根据情绪判断主动联系主人
AI 自主决定哪些群聊内容需要告诉主人
```

## 实现要点

已新增：

```text
src/plugins/ai_chat/owner_notify.py
```

包含：

```text
validate_owner_notification_content()
contains_sensitive_notification_content()
format_owner_notification()
```

主插件中新增命令：

```text
tell_owner_cmd = on_command("转告主人", aliases={"留言给主人", "tell_owner"}, priority=5, block=True)
```

发送给主人私聊时使用 OneBot 主动私聊能力。

需要实际测试 NapCatQQ 是否允许机器人主动私聊主人。

## 验收标准

满足以下条件即可认为第一版完成：

```text
主人发送 /转告主人 测试，主人能收到私聊通知
白名单私聊用户发送 /转告主人 内容，主人能收到来源和内容
授权群成员发送 /转告主人 内容，主人能收到群号、发送人 QQ 和内容
黑名单用户不能转告
非授权群不能转告
陌生私聊用户不能转告
空内容会提示填写 50 字以内内容
超过 50 字固定回复：请主动联系主人，文本过长不予转告。
敏感内容会被拒绝
冷却期间提示转告过于频繁
转告不调用 LLM
转告不写长期记忆
转告失败会记录日志
角色卡内容不影响转告格式
```
