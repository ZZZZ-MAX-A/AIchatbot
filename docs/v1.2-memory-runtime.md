# v1.2 记忆系统运行设计

本文记录当前记忆系统的运行边界和参数。这个版本只推进记忆读取、正式摘要、空窗场景摘要和手动长期记忆；不修改角色卡，不让 AI 自动写入长期记忆。

## 目标

当前记忆系统按下面几层运行：

```text
系统规则
角色卡
手动长期记忆
最近正式会话摘要
空窗场景状态摘要
最近 40 条原文 messages
当前用户消息
```

设计目标：

- 最近 40 条原文保留细节。
- 正式会话摘要保留较早阶段的大意。
- 空窗场景状态摘要桥接“还没正式压缩、也不在最近 40 条”的中间消息。
- 长期事实摘要和主人偏好摘要由主人手动填写。
- 角色卡保持现状，本阶段不自动改写。

## 参数

推荐配置：

```text
MAX_CONTEXT_MESSAGES=40
ENABLE_MEMORY_COMPRESSION=true
MAX_STORED_MESSAGES_PER_SESSION=120
SUMMARY_KEEP_RECENT_MESSAGES=40
SUMMARY_BATCH_MESSAGES=80
SUMMARY_MIN_SOURCE_MESSAGES=40
MAX_SESSION_SUMMARIES_IN_CONTEXT=3

ENABLE_GAP_SCENE_SUMMARIES=true
GAP_SCENE_SUMMARY_1_THRESHOLD=40
GAP_SCENE_SUMMARY_2_THRESHOLD=80
MAX_GAP_SCENE_SUMMARIES_IN_CONTEXT=2

ENABLE_LONG_TERM_MEMORY_CONTEXT=true
MAX_LONG_TERM_MEMORIES_IN_CONTEXT=8
```

核心关系：

```text
120 = 80 + 40
```

含义：

- 未压缩原文超过 120 条后触发正式压缩。
- 最旧 80 条压缩成正式会话摘要。
- 最近 40 条继续保留为原文。
- 正式压缩后清理已覆盖的空窗场景摘要。

## 运行周期

设当前会话未压缩原文数量为 `N`。

### N <= 40

没有空窗。

```text
[最近 N 条原文]
```

### 40 < N <= 80

最近 40 条原文进入上下文，更早的 `N - 40` 条已经离开短时窗口。

系统生成第 1 条空窗场景状态摘要。

```text
[空窗摘要 1]
[最近 40 条原文]
```

### 80 < N <= 120

最近 40 条原文进入上下文，前面最多 80 条形成空窗。

系统生成第 2 条空窗场景状态摘要。

```text
[空窗摘要 1]
[空窗摘要 2]
[最近 40 条原文]
```

当 `N = 120` 时，覆盖关系为：

```text
空窗摘要 1：覆盖第 1-40 条未压缩原文
空窗摘要 2：覆盖第 41-80 条未压缩原文
最近原文：第 81-120 条
```

### N > 120

触发正式压缩：

```text
第 1-80 条 -> 正式会话摘要
第 81-120 条 -> 保留原文
空窗摘要 1、2 -> 清理
```

## 空窗场景状态摘要

空窗场景状态摘要是临时桥接层，不是长期记忆，也不是角色规则。

固定字段：

```text
场景位置：
场景类型：
场景分析：
已明确发生：
当前话题方向：
```

允许记录：

- 明确设定的场景位置。
- 当前对话类型。
- 当前互动氛围。
- 已明确发生或明确确认的事实。
- 后续需要接住的话题方向。

禁止记录：

- 长期回复风格。
- 2-3 句偏好。
- 括号动作描写偏好。
- 称呼偏好。
- 主人心理诊断。
- AI 自己的情绪。
- 未明确发生的亲密行为。
- API Key、Token、密码、二维码、身份证、手机号等敏感内容。

场景位置只记录明确设定。未明确时写“未明确”，不要根据“困了”“洗澡”等词自动脑补地点。

亲密互动只记录明确发生或明确设定的事实，不把未发生的亲密行为写成已经发生。如果主人明确希望角色推进亲密互动，后续回复可以按角色卡和安全边界自然推进，但场景摘要仍不能伪造已经发生的事实。

## 手动长期记忆

长期记忆当前只支持主人手动维护。

系统不会：

- 自动从对话中提取长期事实。
- 自动从对话中提取主人偏好。
- 自动把旧会话摘要迁移到长期记忆。
- 自动修改、删除或合并长期记忆。

长期记忆分两类：

```text
事实摘要：稳定事实，例如项目背景、群聊事实、当前系统长期设定。
偏好摘要：主人长期偏好，例如协作方式、回复风格、明确要求长期生效的习惯。
```

命令：

```text
/添加事实记忆 内容
/添加偏好记忆 内容
/查看长期记忆
/删除长期记忆 记忆ID
```

主体规则：

- 私聊里的事实摘要保存到当前用户。
- 群聊里的事实摘要保存到当前群。
- 偏好摘要保存到当前用户。
- 群聊上下文会读取当前发言用户和当前群的手动长期记忆。
- 私聊上下文会读取当前用户的手动长期记忆。

## 角色卡边界

角色卡仍然作为稳定提示词层参与上下文。

本阶段不做：

- 不修改角色卡文件。
- 不自动把长期偏好写入角色卡。
- 不让 AI 根据对话改写角色卡。

如果需要调整身份、语气、动作描写或角色扮演边界，应由主人手动编辑角色卡或手动添加长期偏好。

## 会话摘要边界

会话摘要仍然由系统自动生成，用于保存当前会话较早阶段内容。

会话摘要适合记录：

- 之前聊过什么。
- 当前任务推进到哪里。
- 已确认事实、决定、纠正、待办和边界。

会话摘要不承担：

- 永久行为规则。
- 长期回复风格。
- 主人长期偏好。
- 角色卡规则。

每次上下文默认只注入最近 3 条正式会话摘要。更早摘要中的长期有效内容应由主人手动提炼到长期记忆，整理后可以删除旧摘要。

## 调试命令

```text
/记忆状态
/摘要状态
/查看摘要
/查看空窗摘要
/查看长期记忆
```

`/摘要状态` 会显示：

- 当前未摘要消息数。
- 自动压缩阈值。
- 保留最近原文数量。
- 每次压缩条数。
- 上下文摘要数。
- 当前空窗摘要数。
- 空窗摘要阈值。

`/查看空窗摘要` 会显示当前会话最多 2 条临时空窗场景状态摘要，包括 slot、覆盖消息数量、覆盖消息 ID 范围、更新时间和摘要正文。

`/记忆状态` 会显示：

- 总消息数量。
- 会话数量。
- 正式摘要数量。
- 手动长期记忆是否注入上下文。
- 长期记忆数量。
- 空窗场景摘要是否开启。
- 空窗摘要数量。
