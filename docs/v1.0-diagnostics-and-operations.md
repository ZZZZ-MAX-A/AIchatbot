# v1.0 稳定性与双通道诊断方案

## 目标

v1.0 的目标是让机器人从“功能可用”进入“可长期运行、可自查、可排错、可维护”的阶段。

v1.0 不把诊断交给聊天 AI。诊断由机器人本体代码执行，输出固定格式结果，不走角色卡，不写入聊天记忆。

## 双通道诊断

诊断分两条通道：

```text
QQ 内诊断命令：机器人还能收发 QQ 消息时使用
本地 PowerShell 诊断脚本：机器人无响应、进程异常或 QQ 链路断开时使用
```

QQ 内诊断不能解决所有问题。如果 NoneBot、NapCatQQ 或 QQ 发送链路已经断开，机器人收不到命令或发不回消息，就必须使用本地诊断脚本。

## 运行依赖

当前完整运行依赖：

```text
1. NoneBot 后端：scripts/start.ps1
2. NapCatQQ / OneBot 接入：scripts/start-napcat-shell.ps1
3. Ollama 本地视觉服务：只在识图时需要
```

关闭任意窗口或进程会影响对应能力：

```text
关闭 NoneBot：机器人逻辑停止，QQ 无响应
关闭 NapCatQQ：机器人收不到 QQ 消息，也发不回 QQ
关闭 Ollama：普通聊天可用，视觉识图失败
聊天 API 不可用：管理命令可用，普通 AI 聊天失败
```

## QQ 内命令

v1.0 第一批主人命令：

```text
/诊断
/配置状态
/视觉状态
/最近错误
/清空错误日志
/图片缓存状态
/清空图片缓存
```

全部命令仅主人可用。

## /诊断

用途：总览机器人当前关键模块状态。

检查：

```text
聊天配置是否完整
聊天接口是否可连通
Ollama 是否可访问
视觉模型是否存在
数据库是否可打开
角色卡是否加载
图片缓存数量
最近错误数量
主要功能开关
```

示例：

```text
诊断结果：
机器人：AI助手
环境：prod

聊天接口：已配置
聊天模型：deepseek-v4-flash
聊天连通性：正常

视觉识图：开启
Ollama：正常
视觉模型：qwen2.5vl:3b
模型存在：是

数据库：正常
消息：128
摘要：4

角色卡：已启用
图片缓存：1 条
最近错误：0 条
```

## /配置状态

用途：只读取本地配置，不联网。

显示：

```text
主人 QQ 是否配置
聊天接口和模型
API Key 是否配置
视觉配置
私聊/群聊/主动回复/主人转告/摘要压缩开关
主要数值配置
明显配置风险
```

不显示 API Key、Token 原文。

## /视觉状态

用途：检查本地 Ollama 和视觉模型。

检查：

```text
ENABLE_VISION
VISION_OLLAMA_BASE_URL
VISION_MODEL
Ollama /api/tags
视觉模型是否存在
图片缓存数量
缓存 TTL
私聊图片等待时间
```

如果 Ollama 不可用，输出错误类型和简短建议。

## /最近错误

用途：读取 `logs/ai_chat_error.log` 最近错误。

输出必须脱敏：

```text
API Key -> [已脱敏密钥]
Token -> [已脱敏密钥]
手机号 -> [已脱敏手机号]
身份证号 -> [已脱敏证件号]
邮箱 -> [已脱敏邮箱]
URL -> [已脱敏链接]
```

不输出完整堆栈和长响应体。

## /清空错误日志

用途：清空 `logs/ai_chat_error.log`。

不影响：

```text
聊天记录
会话摘要
白名单
黑名单
NoneBot 自身日志
```

## /图片缓存状态

用途：查看短期图片缓存概况。

输出：

```text
缓存数量
私聊缓存数量
群聊缓存数量
缓存 TTL
私聊图片等待时间
每轮最多图片
```

不输出图片 URL。

## /清空图片缓存

用途：清空内存中的最近图片 URL，避免旧图误关联。

不影响数据库、摘要、角色卡、权限配置。

## 本地诊断脚本

脚本：

```powershell
.\scripts\diagnose.ps1
```

适用场景：

```text
机器人完全不回复
NoneBot 窗口疑似退出
NapCatQQ 疑似断开
Ollama 疑似未启动
QQ 内诊断命令收不到回复
```

检查：

```text
当前目录
.venv 是否存在
Python 是否可用
bot.py 是否可导入
NoneBot/Python 进程
NapCat/Ollama 进程
11434 端口
8080 端口
.env 关键配置
OLLAMA_MODELS 当前环境
ollama list 中是否有视觉模型
SQLite 数据库是否可打开
最近错误日志
```

## 故障处理顺序

机器人完全没反应：

```powershell
cd D:\AIchatbot
.\scripts\diagnose.ps1
```

机器人能回复管理命令但聊天失败：

```text
/诊断
/配置状态
```

机器人能聊天但识图失败：

```text
/视觉状态
```

群聊图片追问异常：

```text
/图片缓存状态
/清空图片缓存
```

查看近期异常：

```text
/最近错误
```

## 安全边界

诊断命令必须：

- 仅主人可用。
- 不显示密钥、Token、图片 URL、数据库原文。
- 不写入聊天记忆。
- 不触发摘要压缩。
- 不使用角色卡风格。
- 单个模块失败不影响其他模块检查。

## 成功标准

v1.0 第一版完成标准：

```text
主人能在 QQ 内执行 /诊断
主人能确认聊天接口、视觉接口、数据库、角色卡和图片缓存状态
主人能查看和清空错误日志
主人能查看和清空图片缓存
机器人无响应时能运行 scripts/diagnose.ps1
诊断输出不泄露敏感信息
诊断输出不进入聊天记忆
```
